# S&P 500 Directional Forecasting: An Academic Approach

This repository contains the codebase for my thesis exploring the application of deep neural networks in financial time series forecasting. Specifically, it evaluates whether a neural network can reliably predict the next-day directional movement (up or down) of the S&P 500 index.

Predicting the stock market is notoriously difficult due to the incredibly low signal-to-noise ratio. The goal of this project is not to build a holy-grail trading bot, but rather to conduct a scientifically rigorous, academically honest evaluation of deep learning capabilities in highly volatile environments.

### The Architecture

The core model is built using PyTorch and features a Bidirectional Long Short-Term Memory (LSTM) network coupled with an Attention mechanism. 

The pipeline feeds the model a rolling 10-day window of historical data. This data goes beyond simple price action; the feature engineering process calculates bounded technical momentum, trend distances, and most importantly, advanced regime and volatility indicators like the Average True Range (ATR) and the VIX (Volatility Index). 

Because the stock market has a natural upward drift, financial datasets are inherently imbalanced. Rather than artificially balancing the data by throwing away valuable historical days (undersampling), this architecture retains the full dataset. It handles the imbalance mathematically by applying calculated class weights directly into a Binary Cross Entropy loss function. 

### A Note on Academic Rigor and Lookahead Bias

In financial machine learning, it is dangerously easy to achieve an artificial 65% or 70% accuracy by accidentally allowing future data to leak into the training set. 

This codebase was explicitly designed to prevent lookahead bias. The data pipeline strictly enforces causality, ensuring that the model only ever sees information that was historically available at that exact moment in time. Consequently, the performance metrics generated by this model represent a realistic, out-of-sample baseline for directional forecasting, rather than an overfitted statistical illusion.

### How to Run the Pipeline

The project is structured into three distinct sequential stages: data processing, model training, and evaluation. 

**Step 1: Data Preparation**
Run the ETL script to download the raw Yahoo Finance data, calculate the technical and macroeconomic features, and prepare the stationary dataset.

    python src/data/make_dataset.py

**Step 2: Training the Model**
Train the Bidirectional LSTM. The script handles data scaling, chronological sequence splitting, and uses an early-stopping mechanism based on validation loss to prevent overfitting.

    python src/models/train_classifier.py

**Step 3: Evaluation**
Evaluate the saved model against the unseen test set. This generates strict machine learning metrics (Accuracy, Precision, Recall, F1-Score) along with standard academic plots including the Confusion Matrix, ROC Curve, and Precision-Recall Curve.

    python src/models/evaluate.py

### Dependencies

The project relies on a standard Python data science stack. You will need Pandas and NumPy for data manipulation, yfinance for historical data retrieval, scikit-learn for scaling and metrics, Matplotlib and Seaborn for visualizations, and PyTorch for the neural network architecture. 

The scripts include automated hardware detection and will seamlessly utilize CUDA or Apple Silicon (MPS) if available, while safely falling back to standard CPU execution if necessary.
